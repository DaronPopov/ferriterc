=========================
How Ferriterc turns a GPU into a new kind of machine.

=== WHAT NVIDIA PROVIDES (standard CUDA model) ===

  +-----------------------------------+
  |  CUDA Programming Model           |
  |                                    |
  |  cudaMalloc(size) .... ~500 us     |   <-- slow
  |  cudaFree(ptr)                     |
  |  cudaStreamCreate() .. 10-32 max   |   <-- limited
  |  kernel<<<grid,block>>>(args)      |
  |  cudaDeviceSynchronize()           |   <-- full stop
  |                                    |
  |  Mental model:                     |
  |    VRAM = fixed box                |
  |    Fill it. Compute. Read back.    |
  |    If it doesn't fit, give up.     |
  +-----------------------------------+


=== WHAT FERRITERC REPLACES (the software processor) ===

  Every component NVIDIA provides is replaced:

  NVIDIA Component          Ferriterc Replacement           Improvement
  ========================  ==============================  ===========
  cudaMalloc (~500 us)      TLSF O(1) (0.238 us)           4,200x
  10-32 streams             100,000 streams                 3,000x
  Fixed VRAM box            Virtual memory + wave sched     unbounded
  No memory management      VFS + VMM + page swap           OS-grade
  No multi-process          Daemon + SysV IPC               shared GPU
  Manual kernel dispatch    Graph compiler + auto-fusion    optimized
  No autodiff               Tape-based reverse-mode AD      built-in

=== SCALING BEHAVIOR ===

  The processor adapts to any GPU without code changes:

  +--RTX 3070---+   +--RTX 4090---+   +--H100------+   +--B200------+
  | 8 GB VRAM   |   | 24 GB VRAM  |   | 80 GB VRAM |   | 192 GB VRAM|
  | 46 SMs      |   | 128 SMs     |   | 132 SMs    |   | 192 SMs    |
  | SM 86       |   | SM 89       |   | SM 90      |   | SM 100     |
  +------+------+   +------+------+   +------+------+  +------+-----+
         |                 |                 |                |
         v                 v                 v                v
  +------+------+   +------+------+   +------+------+  +------+-----+
  | TLSF pool:  |   | TLSF pool:  |   | TLSF pool:  |  | TLSF pool: |
  |  ~7 GB      |   |  ~22 GB     |   |  ~75 GB     |  |  ~180 GB   |
  | Shard waves:|   | Shard waves:|   | Shard waves:|  | Shard waves|
  |  many small |   |  fewer med  |   |  few large  |  |  minimal   |
  | Streams:    |   | Streams:    |   | Streams:    |  | Streams:   |
  |  same 100K  |   |  same 100K  |   |  same 100K  |  |  same 100K |
  | Alloc time: |   | Alloc time: |   | Alloc time: |  | Alloc time:|
  |  0.238 us   |   |  0.238 us   |   |  0.238 us   |  |  0.238 us  |
  +-------------+   +-------------+   +-------------+  +------------+

  Same code. Same binary. Different GPU. Automatic scaling.

  On bigger GPU:
    - Larger TLSF pool -> bigger shards per wave
    - Fewer waves needed -> less overhead
    - More SMs -> more parallel kernels per wave
    - Alloc latency: unchanged (O(1) is O(1))
    - Stream count: unchanged (100K is enough for anything)
