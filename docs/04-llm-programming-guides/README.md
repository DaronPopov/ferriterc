# LLM Programming Guides

This section defines how an LLM should understand and modify `ferriterc`.

## Files

- `architecture-walkthrough.md`
  Layer-by-layer map for code navigation.
- `change-playbooks.md`
  Standard workflows for feature/addition/refactor tasks.
- `debugging-playbook.md`
  Ordered troubleshooting paths for common failures.
- `safe-edit-rules.md`
  Guardrails for low-risk and high-risk edits.
- `prompting-contract.md`
  Input/output expectations for coding agents operating on this repository.
- `contracts/task-routing-and-gates.md`
  Task-to-subsystem routing and required validation gates.

## Recommended Read Order

1. `architecture-walkthrough.md`
2. `safe-edit-rules.md`
3. `change-playbooks.md`
4. `debugging-playbook.md`
5. `prompting-contract.md`
6. `contracts/task-routing-and-gates.md`
